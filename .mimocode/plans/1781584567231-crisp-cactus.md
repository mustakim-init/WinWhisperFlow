# Model Architecture Overhaul — Production Grade

## Problem Summary

1. **Corrupted GPU model not detected** — turbo ONNX file is 18MB but expects 528MB. No startup health check catches this.
2. **GPU toggle shows wrong models** — only 3 GPU models listed because sherpa-onnx (DML) only supports tiny/base/small. But CUDA (faster-whisper) supports all 6. The UI conflates two different GPU backends.
3. **Model dropdown shows all models** — including undownloaded ones. Should only show downloaded models.
4. **No hardware gating** — AMD user can try CUDA models, wasting time on failures.
5. **No health check at launch** — corrupted models aren't detected or fixed.

## Architecture Decision: GPU Toggle (Recommended)

Keep the **GPU toggle** (not suffix-based names). Reasons:
- VoiceBox uses this pattern — proven UX
- Simpler mental model: "I want GPU acceleration ON/OFF"
- The system auto-selects the best GPU backend (CUDA > DML > CPU)
- Model names stay clean ("small", "turbo") without backend suffixes

**How it works:**
- GPU toggle = preference: "use GPU acceleration when possible"
- When loading a model, the system picks the best backend for that model:
  - CUDA detected + model supports CUDA → use CUDA (faster-whisper)
  - DML detected + model supports DML → use DML (sherpa-onnx)
  - Model doesn't support GPU → fall back to CPU silently
- Models that can't use the current GPU backend show a "(CPU)" badge

## Model Compatibility Matrix

| Model | CPU (faster-whisper) | CUDA (faster-whisper) | DML (sherpa-onnx) |
|-------|---------------------|----------------------|-------------------|
| tiny  | ✓ | ✓ | ✓ |
| base  | ✓ | ✓ | ✓ |
| small | ✓ | ✓ | ✓ |
| medium | ✓ | ✓ | ✗ |
| large-v3 | ✓ | ✓ | ✗ |
| turbo | ✓ | ✓ | ✗ |

**GPU download formats differ:**
- CUDA models: same as CPU (faster-whisper format, auto-downloads)
- DML models: sherpa-onnx ONNX format (separate download via `EnsureGpuModelDownloadedAsync`)

```csharp
private void ValidateDownloadedModels()
{
    string modelsDir = Path.Combine(RuntimePaths.RuntimeRoot, "models");
    if (!Directory.Exists(modelsDir)) return;
    
    foreach (var dir in Directory.GetDirectories(modelsDir, "sherpa-onnx-whisper-*"))
    {
        string modelName = Path.GetFileName(dir).Replace("sherpa-onnx-whisper-", "");
        var onnxFiles = Directory.GetFiles(dir, "*.onnx", SearchOption.AllDirectories);
        
        // Check if ONNX files are suspiciously small (< 10MB for any model)
        foreach (var onnx in onnxFiles)
        {
            var fi = new FileInfo(onnx);
            if (fi.Length < 10 * 1024 * 1024 && fi.Name.Contains("encoder"))
            {
                // Corrupted model — delete it
                AppendLog($"Corrupted GPU model detected: {modelName}. Deleting...");
                try { Directory.Delete(dir, true); } catch { }
                break;
            }
        }
    }
}
```

### 2. Backend: Smart Model Loading (`UIBridge.cs` — `LoadModelAsync`)

Replace `BuildRuntimeOptions` to handle model compatibility:

```csharp
private async Task LoadModelAsync(string model, bool gpu)
{
    // Determine the actual device to use
    string actualDevice = "cpu";
    if (gpu)
    {
        var (provider, _) = GpuDetectionService.DetectCached();
        if (provider is "cuda" or "dml")
        {
            // Check if this model supports the detected GPU backend
            if (IsModelGpuCompatible(model, provider))
                actualDevice = provider;
            else
                Post(new { type = "log", message = $"Model '{model}' not available for {provider.ToUpper()}, using CPU" });
        }
    }
    
    var loadOpts = BuildRuntimeOptions(model, actualDevice);
    // ... rest of loading ...
}

private static bool IsModelGpuCompatible(string model, string provider)
{
    // All models support CUDA (faster-whisper)
    if (provider == "cuda") return true;
    
    // DML (sherpa-onnx) only supports tiny, base, small
    if (provider == "dml") return model is "tiny" or "base" or "small";
    
    return false;
}
```

### 3. Backend: Unified `SendModelsStatus` (`UIBridge.cs`)

Show ALL models with their availability per backend:

```csharp
private void SendModelsStatus()
{
    var (provider, gpuName) = new GpuDetectionService().Detect();
    
    string[] allModels = { "tiny", "base", "small", "medium", "large-v3", "turbo" };
    var models = new List<object>();
    
    foreach (var name in allModels)
    {
        // CPU variant — always available
        bool cpuDownloaded = _whisper.IsModelDownloaded(name, gpu: false);
        models.Add(new
        {
            name,
            displayName = name,
            backend = "cpu",
            backendLabel = "CPU",
            size = GetCpuModelSize(name, cpuDownloaded),
            downloaded = cpuDownloaded,
            loaded = name == _loadedModel && _modelLoaded && _loadedDevice == "cpu",
            available = true,
            gpu = false
        });
        
        // GPU variant — only if GPU detected and model supports it
        if (provider is "cuda" or "dml")
        {
            bool gpuCompatible = provider == "cuda" || name is "tiny" or "base" or "small";
            if (gpuCompatible)
            {
                bool gpuDownloaded = _whisper.IsModelDownloaded(name, gpu: true);
                models.Add(new
                {
                    name,
                    displayName = $"{name} ({provider.ToUpper()})",
                    backend = provider,
                    backendLabel = provider.ToUpper(),
                    size = GetGpuModelSize(name, gpuDownloaded),
                    downloaded = gpuDownloaded,
                    loaded = name == _loadedModel && _modelLoaded && _loadedDevice == provider,
                    available = true,
                    gpu = true
                });
            }
        }
    }
    
    Post(new { type = "models_status", models });
}
```
